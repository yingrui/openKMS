# Knowledge map and home

### Map HTML (LLM overview)

**Regenerate** and **Designer** (`POST /api/knowledge-map/map-html/regenerate`, `…/designer/chat`) share **`_MAP_HTML_VISUAL_DESIGN_APPENDIX`** in **`backend/app/services/knowledge_map_html.py`**: instructs the model to embed SPA-aligned colors, type stack, spacing rhythm, optional dark mode, and distinct styles for hydrated **`a.km-node`** / **`a.km-res`** (see `frontend/src/styles/design-system/_css-variables.scss` — values are duplicated in prose for standalone `<style>` blocks; no `@import`). Designer chats are persisted per user on **`agent_conversations`** / **`agent_messages`** with **`surface=knowledge_map_html`**: **`GET/POST /api/knowledge-map/map-html/designer/conversations`**, **`GET …/designer/session?conversation_id=`**, **`DELETE …/designer/conversations/{id}`**; append after each successful **`…/designer/chat`** completion via **`app.services.knowledge_map_html_designer_session`** (optional **`conversation_id`** in the chat body; omit to use the latest chat). Generated HTML uses placeholder tokens **`{{KNOWLEDGE_MAP_NODE:id}}`** / **`{{RESOURCE:type:id}}`** so **`hydrate_placeholder_links`** keeps working; prompts tell the model to use **Knowledge Map** wording in visible titles/copy.
